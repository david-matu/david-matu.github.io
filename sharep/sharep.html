Server with the SharePoint relying party trust. 

4.  Test the integration: This includes testing the SSO functionality by accessing the 

SharePoint site with a user who is authenticated in the WSO2 Identity Server.

Installing AD FS

 

Hi. How can I get to AD FS in Microsoft Sharepoint 2019? 

To access Active Directory Federation Services (AD FS) in Microsoft SharePoint 2019, you will 
first need to set up AD FS on your network. This typically involves configuring AD FS on a 
separate server and configuring trust between SharePoint and AD FS. 

Once AD FS is set up and configured, you can access it by opening the SharePoint Central 
Administration website and navigating to the "Security" section. From there, you should see an 
option to "configure authentication providers," where you can set AD FS as the primary 
authentication method. 

Keep in mind that this is a high-level overview, and the actual steps to set up and configure AD 
FS with SharePoint may vary depending on your specific environment. 

 

Guide me on the detailed steps 

Here are the detailed steps to set up AD FS with SharePoint 2019: 

1.  Install and configure AD FS on a separate server. This will typically involve installing 

the AD FS role, configuring a federation service name, and creating a trust relationship 
with your SharePoint server. 

Configure a SAML 2.0 Provider for portals with AD FS

2.  Configure the SharePoint server to trust the AD FS server. This can be done by running 

the "Add-SPTrustedIdentityTokenIssuer" PowerShell command, specifying the AD FS 
server's federation service name and a display name for the trusted issuer. 

3.  On the SharePoint Central Administration website, navigate to the "Security" section, and 

click on "Configure authentication providers." 

4.  Select the web application that you want to configure AD FS for, and click on "Default" 

to change the authentication provider. 

5.  Select "Trusted Identity provider" and select your ADFS server from the dropdown. 
6.  Click on "Save" to save the changes. 
7.  After that, you should be able to log in to SharePoint using AD FS credentials.

How do I install AD FS? 

Installing AD FS typically involves the following steps: 

1.  Install the AD FS role on a Windows Server. This can be done by using the Server 

Manager or PowerShell. 

2.  Once the role is installed, open the AD FS Management console to begin the 

configuration process. 

3.  In the AD FS console, click on the "Add Federation Server" link to start the configuration 

wizard. 

4.  Select the option to create a new federation service, and then select the appropriate 

options for your environment. 

5.  After the wizard completes, you will need to create and configure a trust relationship. 

This typically involves configuring the SharePoint server to trust the AD FS server, and 
configuring the AD FS server to trust the SharePoint server. 

6.  Next, you will need to configure authentication methods for AD FS. This can be done by 

using the AD FS Management console to configure authentication providers, such as 
forms-based authentication or Windows authentication. 

7.  Once everything is set up, you can start using AD FS to authenticate users in your 

SharePoint environment. 

Please be aware that these are general steps and some of them might vary depending on your 
environment, network setup, and AD FS version. 

You may also want to check the official Microsoft documentation for further information and 
step-by-step instructions on how to install AD FS.
